# Main 歌词显示区域布局与踩坑记录

本文是 `Main功能交付质量标准.md` 的补充文档，面向 Light 版（原生 WPF 实现）开发者。目标不是要求 Light 逐行复刻 WebView2 实现，而是把 main 当前歌词显示区域的尺寸、比例、边界、动画状态和历史踩坑具象成可对齐的参数。

main 的核心场景是任务栏常驻歌词：空间小、需要快速扫读、不能抢注意力、不能因封面/字体/动画变化造成跳动。Light 可以用 WPF 控件重写，但应对齐这些视觉约束和状态处理策略。

## 1. 当前主布局参数

当前 main 歌词窗口由 WPF 外层窗口和 WebView2 内层歌词页面组成。外层负责贴合任务栏，内层负责封面、歌词、频谱和动画。

| 层级 | 当前参数 | 说明 |
| --- | --- | --- |
| 任务栏基准高度 | `48px` | `AnchorToTaskbar` 中的 `normalTaskbarHeight`。实际任务栏高度取 `max(48, screenHeight - workArea.Height)`。 |
| 歌词窗口高度 | `max(36, taskbarHeight - 4)`，且不超过 `taskbarHeight` | 默认比任务栏少 `4px`，上下居中贴合任务栏。 |
| 外层窗口垂直位置 | `screenHeight - taskbarHeight + ((taskbarHeight - Height) / 2) + YOffset` | 以任务栏区域垂直居中为基准，再叠加用户 Y 偏移。 |
| 外层窗口水平位置 | 左 / 中 / 右锚点 + `XOffset` | 右锚点当前额外预留约 `230px`，避免贴到系统托盘区域。 |
| Web 根容器 `.layout` | `width:100%; height:100%; padding:0 4px; border-radius:8px` | 负责整体背景、圆角、左右安全边距。 |
| 主网格 `.shell` | `grid-template-columns: 34px 8px minmax(0,1fr)` | 封面 34px，中间距 8px，歌词/频谱占剩余空间。 |
| 封面 `.cover` | `34 x 34px`，圆角 `6px` | 不随任务栏高度无限放大，保持小而稳定。 |
| 歌词容器 `.lyrics-pane` | `padding: 3px 4px 0 2px` | 为顶部、右侧和左侧留出文字安全空间。 |
| 歌词层 `.lyrics-layer` | `inset: 3px 4px 0 2px` | 与歌词视觉位置一致。 |
| 频谱层 `.spectrum-layer` | `inset: 0 4px 0 2px; align-items:center` | 与封面垂直居中对齐，不沿用歌词顶部偏移。 |

## 3. 双行歌词尺寸规则

main 的双行歌词不是固定写死两行高度，而是根据实际可用高度动态计算。

| 参数 | 当前规则 | 目的 |
| --- | --- | --- |
| 可测高度 | `viewport.clientHeight - 2px` | WPF host 曾向下扩展 2px 作为 descender buffer，计算行高时排除它。 |
| 单行高度 | `rowHeight = max(13, floor(hostHeight / 2))` | 两行稳定切分，最低保证 13px。 |
| 行间距 | `rowGap = max(0, hostHeight - rowHeight * 2)` | 让两行总高度吃满可用高度，余量作为 gap。 |
| 行距 pitch | `linePitch = rowHeight + rowGap` | 歌词上滑一行时使用的位移单位。 |
| 第一行字号 | `currentSize = min(requestedFontSize, max(11.2, rowHeight * 0.92))` | 避免字号超过行高后上下裁切。 |
| 第二行字号 | `nextSize = max(9, currentSize * 0.92)` | 第二行略小，形成层级。 |
| 第一行透明度 | `0.98` | 接近实色，但避免过硬。 |
| 第二行透明度 | 基础 `0.72`，随进度在约 `0.58 ~ 0.74` 内缓动 | 下一句存在但不抢当前句。 |
| 第一行 Y 偏移 | `1px` | 微调视觉中心。 |
| 第二行 Y 偏移 | `2px` | 让第二行更贴近下方视觉基线。 |
| 行文本底部补偿 | `.line-text padding-bottom: 2px` | 给 `y/j/g` 等下探笔画留空间。 |

Light/WPF 对齐建议：

- 不要只用 `LineHeight = FontSize`。应按“可用高度 -> 行高 -> 字号上限”推导。
- 字体度量要给 descender 留安全区，尤其是 `y/j/g/p/q` 和中文标点。
- 第二行不是简单半透明缩小版；它还要承担“即将进入当前行”的动画起点。
- 字号变化时不要立刻重新排版动画中的层级，应该等当前过渡结束后再刷新度量。

## 4. 歌词切换动画参数

当前 main 的切换动画核心是三行轨道：

```text
currentLine   当前句
nextLine      正在提升为当前句的下一句
incomingLine  新的下一句
```

| 项目 | 当前值 | 说明 |
| --- | --- | --- |
| 位移动画时长 | `560ms` | `.track.animating` 的 transform transition。 |
| easing | `cubic-bezier(0.22, 0.72, 0.24, 1)` | 快速启动、柔和收尾。 |
| 位移距离 | `linePitch * 1` | 每次翻滚移动一行 pitch。 |
| 当前行离场 opacity | `0.98 -> 0.16` | 不是瞬间消失。 |
| 当前行离场 scale | `1 -> 0.972` | 轻微收缩，降低跳变感。 |
| 下一行提升 opacity | 当前第二行透明度 -> `0.98` | 保持层级连续。 |
| 下一行提升字号 | 从第二行字号过渡到第一行字号 | 避免提升瞬间字号跳变。 |
| fallback timer | `transitionDurationMs + 120ms` | 防止 `transitionend` 丢失导致状态卡死。 |

关键状态约束：

- 动画中如果来了新帧，不要立即打断 DOM/WPF 层级，应先放入 `queuedFrame`，当前过渡结束后再处理。
- 切换时必须使用 generation/token，旧封面加载、旧动画回调、旧 timeout 不能覆盖新状态。
- 动画结束时要短暂禁用 transition，交换层级后强制刷新布局，再恢复 transition，避免“长一下又缩回”的反弹。

## 5. 封面显示参数和边界

| 项目 | 当前规则 | 说明 |
| --- | --- | --- |
| 封面尺寸 | `34px` | 与任务栏小高度匹配。 |
| 圆角 | `6px` | 方形但柔和，不做大卡片。 |
| fallback 字号 | `13px`，字重 `700` | 无封面时显示来源字母。 |
| 图片 object-fit | `cover` | 保持填满，不变形。 |
| 图片淡入 | `440ms cubic-bezier(0.16,1,0.3,1)` | 新封面加载后交叉淡入。 |
| 封面 switching 状态 | opacity `0.92`，saturate `0.94`，brightness `0.96` | 加载/切换中微弱反馈。 |
| 封面延迟加载 | `coverSwapDelayMs = 180ms` | 降低频繁切歌时的闪动。 |
| 封面最短可见 | `coverSwitchMinVisibleMs = 420ms` | 避免封面快速连跳。 |

Light/WPF 对齐建议：

- 图片加载必须异步，不要阻塞歌词 tick。
- 旧封面和新封面需要双缓冲或等价机制，不能先清空再加载。
- 加载失败要落到 fallback，不应留白。
- 切歌/封面加载需要 generation，避免上一首封面晚到后覆盖当前歌曲。

## 6. 频谱显示参数

当前 main 频谱用于 Core 传入的纯音乐或特定显示状态。显示层只负责“歌词层”和“频谱层”的视觉切换，不重新判断歌曲是否为纯音乐。

| 项目 | 当前值 | 说明 |
| --- | --- | --- |
| 柱数量 | `21` | 前端 span 与后端 BarCount 必须一致。 |
| 频谱容器宽度 | `min(210px, 100%)` | 不撑破歌词区。 |
| 频谱高度 | `22px` | 贴合任务栏高度。 |
| 柱宽 | `3px` | 小尺度下保持轻盈。 |
| 柱间距 | `3px` | 21 根约占 `21*3 + 20*3 = 123px`，仍有呼吸空间。 |
| 柱圆角 | `999px` | 胶囊形。 |
| 默认静态动画 | `spectrumPulse 1380ms` | 未接入音频时也有轻微生命感。 |
| 音频驱动时 | 关闭 CSS animation，由 JS RAF 平滑 | 避免 CSS 与实时音频争夺高度。 |
| 前端上升速度 | 默认约 `0.56` | 柱高追目标的 rise。 |
| 前端下降速度 | 默认约 `0.24` | 柱高回落的 fall。 |
| 最小柱高 | 默认 `5px` | 暂停/静音时不完全消失。 |
| 高度范围 | 默认 `17px`，设置可调 | 最终高度 = minHeight + level * heightRange。 |

状态约束：

- 频谱层和歌词层不能同时抢视觉，进入频谱模式时歌词层 opacity 降到 0 并上移 `-2px`，频谱层从 `translateY(2px)` 回到 0。
- 暂停时应向前端追加静音目标数组，而不是直接把 DOM 高度硬重置。
- 后端数组长度、前端柱数量、静音数组长度必须保持一致。
- 实时音频推送要做 pending 合并，避免 WebView script 调用堆积。

## 7. 历史踩坑记录

### 7.1 字体度量和“上跳/下跳”

早期尝试处理阿里巴巴普惠体等字体时，歌词在滚动完成后出现上跳、下跳。原因不是单一 CSS，而是字体度量、行高、字号取整和动画结束后的层级交换共同作用。

经验：

- 不要在动画中途重新计算行高/字号。
- 不要依赖某个字体的 ascender/descender 表现，微软雅黑、Segoe UI 和阿里巴巴普惠体差异明显。
- `line-height * 0.92` 这类视觉修正可能短期好看，但对不同字体会放大尾端跳变。
- 保留 descender buffer 比硬压行高更稳。
- 如果某类字体天生度量不适配任务栏小高度，不要为了个别字体牺牲整体稳定性。

### 7.2 `y/j/g` 下探笔画被裁切

第二行歌词里 `y/j/g` 等字母下端曾经视觉偏低或被裁。最终靠 `.line-text padding-bottom: 2px` 和行内 overflow 策略缓解。

经验：

- 只调整行高可能会破坏整体垂直节奏。
- WPF 实现中应关注 `FormattedText`/`TextBlock` 的 baseline 和 descender，而不是只看控件 ActualHeight。
- 英文混排必须作为验收样例，不要只测中文。

### 7.3 动画结束层级交换闪动

如果动画结束时直接把下一行文本复制到当前行，会出现一帧闪烁、字号突然变大或透明度回弹。

经验：

- 交换文本前先关闭 transition。
- 交换后重置 transform、opacity、font-size。
- 强制一次布局刷新，再恢复 transition。
- WPF 中等价做法是：动画完成回调里先冻结视觉状态，再替换绑定内容，最后恢复可动画属性。

### 7.4 快速切歌和旧回调覆盖新状态

封面加载、歌词帧更新、动画 timeout 都可能在切歌后延迟返回。如果没有 generation/token，旧状态会覆盖新歌。

经验：

- 封面、歌词帧、动画都要有当前 track/generation 校验。
- 切歌时要清理 active transition、delayed frame、cover timer。

### 7.5 频谱暂停/恢复不自然

早期暂停时只降透明度，柱高没有回到低位，看起来像“暂停但还悬着”。后来改成给前端持续送静音目标，让柱高按正常速率回落。

经验：

- 暂停/恢复不应另做一套速度，应该复用正常 rise/fall 插值。
- 静音是目标值，不是瞬间重置。
- 恢复播放时也应从当前视觉高度平滑追向新目标。

### 7.6 WebView 调用堆积

频谱更新很高频，如果每帧都 `ExecuteScriptAsync`，WebView 可能积压，实时响应变差。

经验：

- 同一时间只允许一个 spectrum script pending。
- pending 期间只保留最新数组，完成后再发最后一帧。
- WPF 实现如果不用 WebView，也仍然要避免采集线程直接驱动 UI；应通过 UI 帧循环读取最新值。

## 8. Light/WPF 对齐验收清单

Light 每次对齐 main 歌词显示质量时，至少应验证：

- 任务栏底部、高 DPI、多屏、窗口宽度变化后，封面和歌词仍居中且不溢出。
- 默认横向布局满足：封面 34px，封面到文字起点约 10px，左右各有安全边距。
- 中文、英文、数字、`y/j/g/p/q` 混排不裁切。
- 长歌词单行省略自然，不撑高窗口，不挤压封面。
- 当前行/下一行层级明确：字号、透明度、位置有差异。
- 切歌、暂停、恢复、拖动进度时无残影、无错行、无旧封面覆盖。
- 字体、字号、字重切换后不出现完成动画后的上跳/下跳。
- 纯音乐进入频谱时，歌词淡出、频谱居中、封面不动。
- 暂停时频谱柱高平滑回落，恢复时平滑追随。
- 诊断或异常状态不能把普通用户界面变成内部字段堆叠。

## 9. 不建议偏离的边界

- 不要默认做大封面、大卡片或播放器面板式布局。
- 不要把歌词显示区域做成可承载大量状态信息的仪表盘。
- 不要让设置项鼓励用户把窗口调出任务栏可视区域。
- 不要为了单个字体修正破坏其他字体的稳定性。
- 不要让频谱、封面、翻译、歌名歌手同时挤占默认小任务栏区域。
- 不要把 Light 的原生 WPF 实现差异当成降低视觉交付标准的理由。

main 的质量基线可以概括为：小、稳、清楚、有呼吸感。Light 可以更轻，但不应更粗糙。

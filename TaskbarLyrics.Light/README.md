# TaskbarLyrics Light

原生 WPF 轻量版（当前 **v1.2.1**）。Light 版与原版共享 `TaskbarLyrics.Core` 歌词检索逻辑，但歌词窗、设置页、托盘菜单和诊断窗口都使用原生 WPF 实现，不依赖 WebView2 / Chromium 渲染栈，目标是更低进程数、更低内存占用，以及更贴近任务栏常驻场景的交互。

## 与原版对比

### 架构

| 项目 | 原版 (`TaskbarLyrics.App`) | 轻量版 (`TaskbarLyrics.Light`) |
|------|---------------------------|-------------------------------|
| 歌词渲染 | WebView2 + HTML/CSS/JS | 原生 WPF `LyricsDisplayControl` |
| 设置页 | WebView2 加载 `settings.html` | 原生 WPF `SettingsWindow` + `SettingsTheme.xaml` |
| UI 框架 | WPF + WPF-UI | 纯 WPF |
| WebView2 Runtime | 必须 | 不需要 |
| 默认字体 | 内置 Source Han Sans SC | 内置 Source Han Sans SC |
| 歌词业务逻辑 | `TaskbarLyrics.Core` | 共享 `TaskbarLyrics.Core` |
| 歌词缓存 / 歌曲映射 | `%APPDATA%\TaskbarLyrics\` | 与原版共享 |
| 设置与单实例 | 原版独立 | Light 版独立 |

### 性能（历史实测）

测试环境：Windows 11 x64（22631），`Release` 发布，`win-x64` 框架依赖（`--self-contained false`），启动后静置 **20 s** 取样；内存为**主进程及其子进程树**合计（原版含 WebView2）。测试日期：**2026-07-01**；版本：原版 **v1.1.1** / 轻量版 **v1.1.0**。

| 指标 | 原版 v1.1.1 | 轻量版 v1.1.0 | 变化 |
|------|-------------|---------------|------|
| 发布包体积 | 75.7 MB（54 文件） | **67.6 MB**（38 文件） | **约 -11%** |
| 进程数（含子进程） | **7**（1 主进程 + 6x WebView2） | **1** | **单进程** |
| 工作集内存 | **~669 MB** | **~340 MB** | **约 -49%** |
| 专用内存 Private | **~398 MB** | **~234 MB** | **约 -41%** |

> Light 版仍内置思源黑体，所以磁盘体积不会极限缩小；真正减少的是 Chromium 多进程栈、跨进程脚本调用和 WebView2 Runtime 依赖。

## 功能亮点

### 歌词与播放源

- 支持 QQ 音乐、网易云音乐、酷狗音乐、Spotify 的 SMTC 识别
- 播放源可启用/关闭，并支持拖拽调整识别优先级
- 本地歌词支持 `.lrc` / `.qrc` / `.krc`
- 本地歌词策略：
  - 本地优先
  - 在线失败后使用
- 全局歌词偏移，以及 QQ 音乐 / 网易云 / 酷狗 / Spotify 独立偏移
- 数值设置支持直接输入，也支持 `+` / `-` 按钮步进
- SMTC 时间轴稳定器，减少部分播放器时间轴跳变导致的歌词滚动进度错误
- 歌词命中诊断、重新匹配、缓存清理

### 封面

- SMTC 专辑封面异步读取和交叉淡入
- 本地封面回退：
  - 同目录图片
  - 音频内嵌封面
- 封面解码兜底：
  - WPF `BitmapImage`
  - Windows `BitmapDecoder`
  - ImageSharp
- 本地/在线封面策略：
  - 在线优先
  - 本地优先
  - 仅在线
  - 仅本地
- 歌词封面显示开关，默认开启
- 封面样式：
  - 方形
  - 圆角方形
  - 圆形
  - 隐藏封面
- 封面大小调节，横向排列和上下两行布局都生效
- 封面布局：
  - 横向排列：封面在左，歌词/频谱在右
  - 上下两行：封面在上，歌词/频谱在下，并保持下方内容的底部位置不变
- 上下两行布局可单独调整：
  - 歌名/歌手信息显示开关
  - 封面到歌名/歌手信息间距
  - 封面到内容间距
  - 上方封面 X 偏移
  - 上方封面 Y 偏移
  - 歌词/频谱 X 偏移
  - 歌词/频谱 Y 偏移
- 上下两行布局会把封面、歌名/歌手信息、歌词/频谱和 X 偏移一起纳入自动宽度计算
- 可用封面主色给歌词和频谱取色

### 频谱

- 24 条实时频谱
- 频谱总开关
- 纯音乐时显示频谱
- 未检索到歌词时显示频谱（默认开启）
- 有歌词时也显示频谱（默认关闭）
- 频谱样式：
  - 中心扩散
  - 底部柱状
  - 镜像波形
  - 细线
  - 点阵
  - 呼吸条
- 频谱调参窗口，可调整响应速度、高度范围、透明度和刷新间隔
- WASAPI 环回采集按需启动，仅在需要显示频谱时采集音频

### 窗口与动画

- 双行歌词显示
- 歌词切换动画：
  - 上滑
  - 淡入淡出
  - 紧凑滑动
  - 无动画
- 使用 `CompositionTarget.Rendering` 驱动歌词过渡和频谱插值，动画更贴近屏幕刷新
- 上下两行布局会为歌词切换动画保留垂直裁剪缓冲，减少第二行上滑时文字只显示半截的问题
- 自动窗口宽度，随歌词内容变化；上下布局会额外计算封面、歌名/歌手信息和 X 偏移
- 自动窗口高度，随字号、封面布局和歌词区域变化；上下两行布局增加高度时优先向上展开，底部锚点不变
- 可手动设置窗口宽度、高度、自动尺寸偏移
- 水平锚点：左侧 / 居中 / 右侧
- X / Y 偏移
- 强制置顶开关
- 从 Alt+Tab 隐藏（`WS_EX_TOOLWINDOW`）
- 背景显示开关、背景不透明度、背景材质、边框、文字阴影

### 托盘与启动

- 单实例，二次启动会激活已有实例并打开设置
- 系统托盘菜单
- 开机自启动（当前用户注册表 `Run` 项，独立名称 `TaskbarLyrics.Light`）
- 播放器联动：
  - 检测到已启用播放器开始播放时自动显示歌词
  - 全部播放器停止或关闭后自动隐藏歌词
  - 启动阶段若开启“播放器关闭时隐藏”，默认不闪出歌词窗

### 更新检查

- 关于页支持检查 GitHub Release
- 优先使用 GitHub API
- API 失败时回退到 `releases/latest` 跳转地址解析，降低部分网络环境下误报“网络不行”的概率
- 自动检查更新每天最多一次

## 设置页结构

Light 版设置页已按功能域重排，后续新增功能请按下面规则放置：

| 分区 | 放置内容 |
|------|----------|
| 播放源 | 播放器启用、播放器图标、识别优先级 |
| 歌词与本地库 | 启动显示、播放器联动、歌词偏移、本地歌词、本地封面、本地目录 |
| 显示与外观 | 频谱、字体颜色、封面样式、封面布局、背景、边框、阴影、切换动画 |
| 窗口布局 | 自动尺寸、手动宽高、锚点、X/Y 偏移、置顶 |
| 维护调试 | SMTC 监视器、频谱调参、歌词诊断、重新匹配、缓存清理 |
| 关于更新 | 版本信息、自动检查更新、手动检查更新、仓库入口 |

## 性能优化

- 延迟初始化 SQLite / EF Core 与歌词同步服务
- 在线歌词 Provider 懒加载
- 统一 16 ms 前端帧定时器，歌词和频谱按不同节奏调度
- 频谱按需启动 WASAPI 环回采集
- 行高与文本宽度测量缓存
- WPF 画刷和封面 Bitmap 冻结，减少跨线程和渲染开销
- 设置页字体列表延迟枚举
- 封面读取失败会记忆拒绝签名，避免同一坏图反复解码

## 当前默认值

Light 版 1.2.1 的恢复默认设置 / 首次启动默认偏向当前推荐的常驻使用效果：

- 未检索到歌词时显示频谱：开启
- 有歌词时也显示频谱：关闭
- 前景色：自定义橙色 `#FFFF8000`
- 跟随封面取色：开启
- 文字阴影：开启
- 上下两行布局歌名/歌手信息：开启（仅切到上下两行布局时显示）
- 默认窗口 Y 偏移：`-40`

## 系统要求

- Windows 10/11 x64
- 从源码运行或框架依赖发布时，需要安装 [.NET 8 Runtime](https://dotnet.microsoft.com/download/dotnet/8.0)
- 不需要 Microsoft Edge WebView2 Runtime

## 运行

```bash
dotnet run --project TaskbarLyrics.Light/TaskbarLyrics.Light.App
```

## 发布

框架依赖发布（需目标机器已安装 .NET 8）：

```bash
dotnet publish TaskbarLyrics.Light/TaskbarLyrics.Light.App -c Release -r win-x64 --self-contained false -p:DebugType=None -p:DebugSymbols=false -o publish/light
```

独立发布（无需目标机器预装 .NET 8）：

```bash
dotnet publish TaskbarLyrics.Light/TaskbarLyrics.Light.App -c Release -r win-x64 --self-contained true -p:DebugType=None -p:DebugSymbols=false -o publish/light-standalone
```

发布包请完整解压后运行 `TaskbarLyrics.Light.exe`，需要保留 `Assets` 等目录。

## 配置与数据目录

| 用途 | 路径 |
|------|------|
| Light 设置 | `%APPDATA%\TaskbarLyrics.Light\settings.json` |
| 单实例锁 | `%LOCALAPPDATA%\TaskbarLyrics.Light\TaskbarLyrics.Light.lock` |
| 运行日志 | 程序目录下 `Logs\app_debug.log` |
| 歌词缓存 | `%APPDATA%\TaskbarLyrics\cache`（与原版共享） |
| 歌曲映射数据库 | `%APPDATA%\TaskbarLyrics\database\song_maps.db`（与原版共享） |

两版可以同时安装；设置文件和单实例锁独立，歌词缓存与歌曲映射数据库共享。

## 选用建议

- 优先 Light 版：日常任务栏常驻、在意内存和进程数、不想依赖 WebView2、需要开机自启、需要播放器联动、只显示原文歌词
- 优先原版：需要歌词翻译，或希望完全跟随主仓库 WebView2 设置页体验
